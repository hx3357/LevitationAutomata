## filter系统

只有符合filter规则的消息会输入agent。实现时注意每个步骤解耦合。

1. 如果消息发送人命中黑名单，丢弃
2. 监听聊天范围条件
    1. 未命中聊天表，丢弃。
3. 命令处理条件（前提：消息可解析为 slash 命令，否则跳过本组）
    1. 按命令名取其权限级别：管理员命令 / 普通命令。
        1. 管理员命令：要求**发送人命中管理员list 且 第一个参数（TOTP）校验通过**（混合方案，见“管理员鉴权”）。
            - 任一不满足（未命中 list 或鉴权失败）：不执行，通知 agent「{发送人}试图使用非法指令」，日志 warning。
            - 两者均满足：执行处理函数，通知 agent「{发送人}使用管理员指令 {指令名}：{指令内容}」。
        2. 普通命令：执行处理函数，通知 agent「{发送人}使用指令 {指令名}：{指令内容}」。
        （“通知 agent”为只读旁白事件，命令不进 agent 解释。）

## 命令系统

/<命令名> <鉴权参数> [位置参数...] [--选项 值]
   └─ 必填   └─ 仅管理员命令需要(TOTP 动态码)

### 普通命令

返回的默认渠道是wx platform

1. /weather [位置]  拉起子agent，继承主agent的system prompt，查询天气，通过wx platform返回结果
2. /cron [内容] 拉起子agent，继承主agent的system prompt，根据用户的内容设置cron job。如果必要信息（时间，动作）不全或者内容无关，返回反馈。最后返回结果。
3. /help 返回所有命令，介绍和参数含义

### 管理员命令

默认具有<鉴权参数>，声明中省略

1. /restartwx 重启wxauto模块（刷新wechat类），通过wx platform返回结果
2. /statwx 通过wx platform返回wxauto模块状态和参数
3. /ban <username> 把username用户加入黑名单

#### 管理员鉴权

由于`sender`字段只是显示名、可被群成员改昵称伪造，因此命中管理员list不足以授权，必须叠加一道与“管理员鉴权密钥”绑定的鉴权。采用 **TOTP（基于时间的一次性口令，RFC 6238）**，即 Google Authenticator 同款方案。最终采用list命中+鉴权通过混合方案。

规则：

1. **共享密钥**：配置模块中的“管理员鉴权密钥”即 TOTP 共享密钥，管理员侧用同一密钥生成动态码。
2. **鉴权参数**：每个管理员命令的第一个参数固定为当前 TOTP 动态码（6 位数字，30 秒周期）。
3. **校验**：服务端用同一密钥与当前时间生成期望码，与第一个参数做 **constant-time 比较**（`hmac.compare_digest`）；为容忍时钟漂移，允许相邻 ±1 个时间窗。
4. **防重放**：维护一个已用动态码的短期缓存（TTL ≈ 一个时间窗）。同一动态码在有效期内只接受一次，命中缓存即判为失败。这弥补了群聊中命令对所有成员可见、可在窗口内复制粘贴重放的风险。
5. **失败处理**：鉴权失败按 filter 命令处理条件中“鉴权参数错误”分支处理（不执行命令），同时在日志输出 warning，记录“管理员名 + 错误动态码”作为潜在冒充/爆破信号。

实现建议使用 `pyotp` 库（`pyotp.TOTP(secret).verify(code, valid_window=1)`），无需自行拼接时间字符串或做可逆加解密。

##### 添加管理员的流程

混合方案下，授权 = 命中管理员list **且** 鉴权通过。所有管理员共享同一“管理员鉴权密钥”，新增管理员**无需改动密钥或服务端登记**：

1. 将该管理员的显示名（`sender`）加入配置模块的“管理员list”。
2. 通过**私密线下渠道**把“管理员鉴权密钥”（TOTP 的 Base32 密钥）交给该管理员——切勿在受监听的微信聊天内发送。
3. 管理员在任一 TOTP 应用（Google Authenticator、1Password、`pyotp` 等）中录入该密钥：直接输入 Base32 字符串，或扫描由密钥生成的 `otpauth://totp/...?secret=<KEY>` 二维码。
4. 录入完成后，该管理员即可生成动态码。两个条件缺一不可：未入 list 即使有正确动态码也不授权，入了 list 没有动态码同样不授权。

##### 管理员执行命令的操作步骤

1. 打开 TOTP 应用，读取当前 6 位动态码。
2. 按命令格式把动态码作为**第一个参数**：`/<命令名> <动态码> [其余参数...]`。
   例：`/ban 837461 网友塔顺菲`
3. 在命中聊天表的聊天里发送该命令。
4. 动态码 30 秒有效且**一次性**：过期或已用过需用新码重发，鉴权失败会收到“试图使用非法指令”。群聊中该码虽对所有成员可见，但因一次性 + 短时效 + 防重放缓存，无法被他人重放。
